║ 1. Project Overview ║
╚════════════════════════════╝
Full-featured ECMS to control all aspects of online business from one central dashboard.
Optimized for Cash on Delivery (COD) and online payments.
Includes modules for orders, products, inventory, customers, suppliers, finance, campaigns, landing pages, reports, abandoned orders, and customer risk analysis.
Single Vendor Mode:
Managed by a single admin.
Full access to products, orders, campaigns, customers, and accounts.
Vendor-specific features hidden.
Multi-Vendor Mode:
Supports multiple vendors selling through a unified platform.
Each vendor has their own dashboard for products, orders, and reports.
Admin controls vendor approval, commissions, and platform-wide settings.
Goals:
Centralized business control
Smart order management with real-time stock tracking
Customer behavior monitoring and fake order prevention
Abandoned order recovery
Accurate profit/loss calculation
Scalable and secure architecture

╔════════════════════════════╗
║ 4. Order Module ║
╚════════════════════════════╝
Add Order Workflow:
Customer Information: Name, Phone, Alternative Phone, Address, City, Sub City
System checks existing orders, shows success rate and risk badge
Product Selection: Search, select variant, quantity, auto subtotal
Pricing: Product price, discounts, delivery charge by city, total calculation
Order Details: Source, payment method, courier, internal notes
Save Order: Reduce stock, generate invoice, status Pending
Order Status System:
Statuses: Pending → Confirmed → Processing → Shipped → Delivered → Cancelled → Returned
Timeline logging for each admin action
Courier API Integration:
Automatically generate consignment IDs, store tracking, print labels
Multi-Vendor Specifics:
Orders can include multiple vendors' products
Split payment calculation: vendor revenue + platform commission
Stock updates per vendor
Vendors see only their orders
╔════════════════════════════╗
║ 5. Abandoned Order System ║
╚════════════════════════════╝
Triggered when customer leaves before placing order
Data captured: Name, Phone, Address, Products, Quantity, Total, Date, Source
Admin actions: Call, convert to order, follow-up, delete, add notes
Dashboard displays abandoned orders with filters
Multi-Vendor: vendors see only their own abandoned orders
╔════════════════════════════╗
║ 6. Customer Risk Analysis ║
╚════════════════════════════╝
Tracks: Total Orders, Delivered, Returned, Cancelled, Exchange, Revenue, Last Order Date
Delivery Success Rate: (Delivered Orders ÷ Total Orders) × 100
Risk Classification: Trusted (80%+), Medium (60–79%), High (40–59%), Blacklisted (<40%)
System shows risk badges during Add Order, Order List, Customer Profile
Optionally block blacklisted customers to prevent fake orders

║ 23. Super Admin Mode Control ║
   
╚════════════════════════════╝
Super Admin assigns client access mode:
Single Vendor: client sees only single-vendor modules
Multi-Vendor: client sees full multi-vendor marketplace modules
Mode switching:
Multi → Single: all vendor modules hidden/deactivated
Single → Multi: vendor modules activated
Super Admin dashboard allows viewing clients, assigning mode, tracking activity, and monitoring sales
